# Sistema-Electoral-Estudiantil

Objeto de Estudio: 

Que el estudiante implemente todo el conocimiento adquierido hasta la fecha en la creacion de un proyecto de valor, y de paso refuerce los conocimientos ya adquiridos.

Caso de Aprendizaje: Creacion de un sistema de Votaciones estudiantiles.

Debe usar Programación orientada a objectos para la creación de las entidades y cálculos pertinentes.

Debe usar una separación en capas para que el código sea mas limpio y ordenado.

de preferencia MVC.

Puede usar una base de datos de su preferencia como SQL server, maría DB o PostgreSQL.

Debe tener un entorno visual agradable a la vista y fácil de entender.

Debe ser manejado con Git y GitHub.

Crear una aplicacion en c# para llevar a cabo las votaciones escolares a nivel de colegio, y o de curso, este sistema necesita Registrar las votaciones, tener las estadisticas de la votaciones, tener el registro de los participantes, llevar el control de las planchas, el registro de los usuarios entre otros.

Debe de contar con al menos los siguientes modulos:

Modulo #1 votaciones:

Aqui Saldran los candidatos o las planchas y el usuario votara por un candidado una plancha o un Nulo.

Modulo #2 Menu:

Aqui ira el menu principal con las principales funciones del sistema.

Modulo #3 Datos Generales de la votacion (Datos del Partido):

Aqui Saldran datos como Votos Emitidos, Padron Electoral, participantes, Totales, Tiempo Restante de Votacion, Posicion electoral de los partidos.

Modulo #4 Planchas Electorales:

Aqui se crearan o conformaran las planchas eletorales y deben tener informacion como los candidatos y puestos, Logo del partido, etc.

Modulo #5 Registro de Usuarios:

Aqui se mostraran informaciones de de los usuarios y se llevara a cabo el registro de los mismos como votantes o administradores de un partido, se registraran informaciones como el nombre,el curso, la seccion, la matricula, etc.

Modulo #6 Login:

Aqui se iniciara la seccion de los usuarios con roles especificos.

Modulo #7 Panel de votaciones:

Cada vez que se realiza una votacion este panel se debe de actualizar aqui se debe reflejar el Porcentaje de Votos Emitidos, El porcentaje de Votos Nulos, El porcentaje de personas que an llevado a cabo el voto.

Reglas del Negocio:

Un usuario no puede votar mas de dos veces.

Las votaciones no pueden ser amañadas, o manipuladas por tercero.

Un votante de un padron X no puede ejercer en otro padron. 

La votacion es secreta un partido podra saber si un participante voto mas no por quien.

Reportes del Sistema:

-Reporte de plancha ganadora con sus estadisticas.

-Reporte de los integrantes del partido.

-Reporte General de votos

-Reporte listado generales como los de participantes o votantes.
